1). Moviesstore Description

This web store pretty much enables users to have an all-inclusive experience when buying and reviewing movies. We have 5 different tabs that exist: About, Movies, Cart, Orders, and Sign-Up. The about page simply just states what the website is about. There is no interactive experience for the user on this page. The next pages is Movies where we see an array of things. First, you will see a list of movies. Here, there are only four, but using the search feature, you can parse through to find the movie you want. If you click on a movie, you can see more about it, including its description, price, and reviews. You can add reviews here, edit them, and later delete them. The next page is the Cart page where it shows all the movies in your shopping cart. On the bottom, you can see your total, the ability to clear your cart, and finally you can buy your movies if so. The page is Orders, which shows you all the orders you've made from your account and their timestamp. The last page is just the Sign Up interface where you can make an account, log in, log out to keep track of your account.

2). Process

  • Setup: I first had to upgrade my python since it was compatible to complete this tutorial. Additionally, I had to dowload django since I had never previously worked with it before.
  • Implementation: For this project, I worked in an iterative approach working page by page. From my base django, Consequently, for each page, I followed a similar sequence of steps. I first added url patters to route to the new page, and then added page specific templates (inheriting from a base skeleton page). I then defined models to describe the artificats that would be used in the site, and then levereged Django's migrations to create and mangage the database tables via SQLite.
  • Challenges: The primary challenge I ran into was dealing with PyMySQL. While attempting to connect my database to PythonAnywhere, I kept receiving error prompts that MySQL could not reach my server. To resolve this, I simply just went back to SQLite, but in the future, I want to actually use MySQL to make my stack more efficient and thorough.
  • Deployment: Essentially, to deploy, I pushed my code using Git, and used PythonAnywhere to set up a virtual environment, configure the WSGI file, and finally making the web app live.
